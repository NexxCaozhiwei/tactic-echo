# Tactic Echo 1.0.05 基线

## 自动爆发前置注入与窗口展示修复

1.0.05 是 1.0.04 后针对实机“前置 `复仇之怒（31884）→ 处决宣判（343527）`”问题的修订基线。

- **共享 GCD 归类**：`IconState` 在技能冷却与全局 GCD 的开始、持续和剩余时间严格对齐时，将短冷却归类为共享 GCD；AutoBurst 因此等待 `GCDGate` 的 `QUEUE_WINDOW`，不会把已就绪注入技能当作自身 CD 跳过。
- **观察帧 HUD 绑定**：Burst 的等待/窗口离开锁帧继续保持 `BindingToken=0` 且 TEK 不输入，但 SignalFrame 同时提供只读官方展示绑定，HUD 不会把官方 `处决宣判` 误显示为“未绑定”。
- **窗口重复保护**：窗口步骤已候选派发或确认后，官方推荐仍短暂显示同一窗口时，AutoBurst 维持 `armed + observationOnly` 直到推荐离开，防止重复窗口键；未派发窗口的中止计划则立即归还普通官方派发。
- **战斗边沿**：`PLAYER_REGEN_DISABLED` 开始新的 combat epoch，清除上一战斗遗留的窗口锁与最后官方推荐；当前战斗的首次健康窗口推荐可正常触发。
- **边界不变**：不绕过 AutoBurst 双开关、动作条 BindingToken、TEAP、TEK、确认、限频、前台或人工接管门禁；Phase 1 仍只支持一条显式窗口/注入规则与单一用途宏。

历史基线只用于追溯，不作为新的开发起点。
