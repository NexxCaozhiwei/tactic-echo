# P4.5 整合打断宏实机验收

## 1. 宏与键位

将以下宏放到当前可见的 Blizzard 默认动作条，分配一个实际安全键位：

```lua
#showtooltip 迎头痛击
/stopcasting
/cast [@mouseover,harm,nodead][@focus,harm,nodead][harm,nodead] 迎头痛击
```

以你的实际职业打断技能替换“迎头痛击”。宏不能检测读条；其 Blizzard 优先级始终是 mouseover、focus、target。

## 2. 前置条件

- 覆盖 AddOn 后 `/reload`；
- TE 处于 `armed` 且在战斗中；
- 自动爆发没有活动 plan / pre-window capture；
- 自动打断中已启用拟测试来源；
- 打断设置中识别到 `宏，鼠标指向→焦点→当前目标；仅同一宏分支可命中时自动可按`。

## 3. 用例

| 场景 | 预期 source / 结果 |
|---|---|
| mouseover 正在读可打断条 | `source=mouseover`，`routeMode=macro_priority_chain`，一次 reaction 按键 |
| focus 正在读；mouseover 为空、友方或死亡 | `source=focus`，一次 reaction 按键 |
| focus 正在读；mouseover 为另一只活敌但未读条 | `macro_priority_preempted_by_mouseover`，零按键 |
| target 正在读；focus 为活敌但未读条 | `macro_priority_preempted_by_focus`，零按键 |
| target 正在读；mouseover 和 focus 均无效 | `source=target`，一次 reaction 按键 |
| 钢条或未确认读条 | P4.4 原有硬否决/未确认，零按键 |

## 4. 采集字段

从 `TacticEcho.lua` 的 `autoReactionRuntime.last` / `events` 观察：

```text
reason
source
routeMode
macroPriorityChain
routeReason
confirmationReason
bindingToken
castKey
```

成功候选应包含：

```text
reason = confirmed_interrupt_candidate
routeMode = macro_priority_chain
macroPriorityChain = true
```

拦截场景应包含：

```text
reason = macro_priority_preempted_by_mouseover
# 或
reason = macro_priority_preempted_by_focus
```
