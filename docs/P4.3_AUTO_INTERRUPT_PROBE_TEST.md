# 1.0.40 P4.3 自动打断动作测试

本补丁是临时探测版：它会忽略钢条/未确认状态，以确认自动打断动作能否抵达 TEK 与游戏。只建议在可控的训练假人、低风险战斗或可接受误按的场景中测试。

## 安装

1. 退出游戏并停止当前 `TEK.exe`。
2. 将本压缩包直接解压到现有 Tactic Echo 项目根目录，选择覆盖同名文件。
3. 在项目根目录双击 `TEKEXEBUILD.CMD`，等待生成新的 `dist\TEK.exe`。
4. 用新的 `dist\TEK.exe` 替换/启动当前 TEK，随后启动游戏并 `/reload`。
5. 在 TE 的“打断设置”中确认“自动打断”已开启，至少勾选“当前目标”，并确认打断技能/宏显示为安全可用路由。

## 预期现象

- 任何敌对、存活、真实读条的当前目标都会触发一次自动打断动作尝试，即使它是钢条。
- SavedVariables 的 `TacticEchoDB.autoReactionRuntime.last` 应出现 `state = "candidate"` 及 `reason = "probe_interrupt_candidate"`。
- TEK Trace 首次应显示 `dispatch_origin = reaction` 与 `input_sent`；同一读条持续时，重复帧应为 `reaction_candidate_already_dispatched`，不会连续按键。

## 测试后请反馈

请提供同一次读条附近的：

- `TacticEchoDB.autoReactionRuntime.events` 最后 10 条；
- TEK Trace 中 `dispatch_origin=reaction` 或 `reaction_candidate_already_dispatched` 的记录；
- 是否实际按出打断技能，以及是否只按一次。
