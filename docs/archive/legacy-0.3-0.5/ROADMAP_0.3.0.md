# 0.3.0-next 路线图

## 目标

将已具备基础链路的 Tactic Echo 从“协议/输入验证 POC”推进为一个可日常使用、可看懂、可诊断、可部署的惩戒骑辅助工具。

## 里程碑 A：TEK.exe 产品化（当前优先）

### 范围

- `TEK.exe` 唯一用户入口；
- 托盘常驻、浅色高对比状态图标；
- 默认 UI 联动等待；
- 无预算持续处理有效 freshness；
- 暂停/恢复/停止/退出语义；
- 常规 / 连接 / 诊断设置；
- 自动定位缓存、固定坐标回退；
- 单实例、日志、错误通知、PyInstaller 打包；
- 全量自动测试与 Windows smoke test。

### 完成标志

- 用户无需终端即可使用 TEK.exe；
- Windows 启动会话首次打开状态窗口；
- 默认 UI linked worker 能正确等待 WoW 和 TE 武装；
- Pause/Stop/Exit 行为可靠；
- Blocked 自动恢复、ErrorLocked 显式恢复；
- 无 Python Windows 环境可运行打包产物；
- 自动测试全绿，托盘人工验证完成。

## 里程碑 B：爆发建议

### 范围

- 惩戒骑职业大招建议；
- 主动饰品建议；
- 爆发药水建议；
- 战前增益缺失提醒；
- 爆发窗口、理由、阻断码与每窗口上限。

### 不在范围

- 同帧多动作输入；
- 组合宏；
- 未验证物品的真实输入；
- 自定义职业循环替代官方推荐。

### 完成标志

- BurstPlan 可在 UI/trace 中解释每条建议；
- 不改变官方主动作；
- 每个辅助候选有唯一 windowId 与阻断原因；
- 自动测试覆盖窗口去重和策略开关；
- 至少一个真实木桩场景完成“建议合理性”人工检查。

## 里程碑 C：游戏内实用 UI

### 战斗页

显示当前推荐、键位、TE 状态、绑定摘要、爆发窗口摘要和武装/暂停控制。

### 爆发页

显示爆发开关、职业大招/饰品/药水建议、战前增益提醒、窗口/理由/阻断码。第一版全部为建议模式。

### 设置页

显示 Profile、绑定、位置/缩放、简洁模式、信号/安全说明和高级诊断入口。

### 完成标志

- 默认 UI 不再出现大段协议字段和日志；
- 用户能理解每个按钮的影响范围；
- 调试信息仍可通过高级入口获取；
- UI 配置能安全持久化并在损坏时回退默认值。

## 推荐实现节奏

```text
TEK 单实例 + 测试修复
→ UI linked 默认生命周期 + 无预算连续处理
→ 设置/定位/图标/打包 Windows 验证
→ Burst 数据层 + 单元测试
→ Burst 建议 UI
→ 游戏内战斗/爆发/设置 UI
→ 单项人工效果验证
```
