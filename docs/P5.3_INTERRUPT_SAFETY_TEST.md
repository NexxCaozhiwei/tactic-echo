# P5.3 实机验收：钢条稳定证据、打断冷却门控与目标管理宏守卫

## 安装

1. 覆盖项目根目录补丁文件。
2. 覆盖 WoW 实际加载目录 `Interface/AddOns/!TacticEcho`。
3. 重启 TEK；本版没有 TEK 代码改动，不需要重建 `TEK.exe`。
4. 进入游戏后执行：

```text
/reload
/tecache refresh
/temapping p53_interrupt_safety
```

确认“启用自动打断”和“兼容未确认读条派发”开启；打断按钮或已有打断宏必须放在当前可见 Blizzard 默认动作条并有真实键位。

## 场景 A：普通 opaque 读条仍可进入 P4.3 链

对一个普通可打断读条、但面板仍显示 `interruptibility_unconfirmed` 的目标：

```text
第 1 次：compatibility_evidence_settling / compatibilityEvidenceSamples=1
第 2 次：compatibility_evidence_settling / compatibilityEvidenceSamples=2
第 3 次：compat_interrupt_candidate / compatibilityEvidenceSamples=3
```

第三次后才允许：

```text
dispatchOrigin = reaction
bindingToken > 0
```

TEK trace 的 `dispatch_origin=reaction` 仍只应对同一稳定 sequence 成功输入一次。

## 场景 B：真实钢条不得进入 compatibility candidate

用同一来源测试明确不可打断读条。预期为以下之一：

```text
native_visible_shield
unit_api_not_interruptible
unit_event_not_interruptible
native_not_interruptible
compat_native_steel_suspected
interruptibility_unconfirmed
```

不论具体证据来源，均不得有：

```text
reason = compat_interrupt_candidate
dispatchOrigin = reaction
reaction BindingToken > 0
```

保存 `autoReactionRuntime.last.nativeEvidence`、`eventEvidence`、`compatibilityEvidenceSamples` 和面板中的“盾组件/原生/兼容采样”。

> 限制：若该客户端对某个钢条既不暴露可见盾、也不发送 `NOT_INTERRUPTIBLE`、也没有 scalar/method 嫌疑，AddOn 无法仅凭 unknown 可靠区分它。此时上传日志，不应把未观测到的证据当成“已经识别钢条”。

## 场景 C：打断技能冷却时必须释放主键

1. 先手动使用打断技能，使其进入自身 CD；
2. 立刻让敌人开始读条；
3. 检查面板与 SavedVariables：

```text
reason = interrupt_action_cooldown
routeReason = p2_route_ready_but_action_cooldown
cooldownActive = true
cooldownOnGCD = false
cooldownActionSlot = <实际打断按钮槽位>
bindingToken = 0
```

主键必须恢复官方正常推荐，不能保持反制射击/打断技能。纯 GCD 不得被误判为打断 CD。

4. CD 结束后，下一段读条在兼容采样 `3/3` 后应重新进入 `compat_interrupt_candidate`。

## 场景 D：目标管理宏

对于：

```lua
#showtooltip
/stopcasting
/cast [@focus,nodead] 反制射击
/cleartarget
/targetenemy
/cast 反制射击
/targetlasttarget
```

- 焦点敌人读条：可使用该已有宏的焦点分支；
- 当前目标读条：不得自动使用后续 `/targetenemy` 分支。预期为：

```text
macro_managed_target_target_unverifiable
```

或精确解析形态的历史原因：

```text
macro_managed_target_fallback_target_unverifiable
```

当前目标自动打断应改用普通当前目标技能按钮或无目标管理命令的严格 target 宏。
