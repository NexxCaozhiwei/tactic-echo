# P5.9 控制、防御与生存宏兼容实机验证

先在当前可见 Blizzard 默认动作条上分别准备下列**已有宏**；不要让 Tactic Echo 创建、复制或修改宏：

- 控制：含 `@focus`、`@mouseover`、`@cursor`、条件分支或目标管理辅助命令的控制技能宏；
- 防御：含 `/cast`、`/stopcasting`、条件分支、`@player` 等既有防御宏；
- 生存：`/use 5512`、`/use item:5512`、`/use [@player] 治疗石`，以及可选的条件/多分支或 `/castsequence` 消耗品宏。

1. 在“按键按下即施放（ActionButtonUseKeyDown）”分别开启、关闭时，触发控制、防御、生存 HUD 卡。HUD 图标应保持正常显示；只要对应宏位于当前可见默认动作条，点击 HUD 必须与直接点击该原动作条宏完全一致。
2. 移除宏的快捷键但保留其可见动作条按钮。控制、防御、生存 HUD 卡仍应可手动点击；诊断/来源可显示无标准键位，但不得因此消失。该来源的 `BindingToken` 必须为 `0`，不得出现 TEAP/TEK 控制、防御或生存派发。
3. 对控制、防御宏分别验证 `@focus`、`@mouseover`、`@cursor`、条件分支、`/castsequence` 与目标管理辅助命令。Tactic Echo 不得拆分或评估分支，也不得切换/恢复目标；实际结果必须完全由原宏决定。
4. 对生存物品分别验证 ItemID、`item:ID` 与本地化物品名。可再验证含两个物品分支或 `/castsequence` 的既有宏：HUD 只复用该宏本体，不能替换为单独物品、不能猜测当前会走哪一分支。
5. 令对应按钮隐藏、切换到特殊动作条、或在战斗中改变动作条页/宏来源。HUD 卡应继续显示但阻断点击，并给出原因；离开战斗后刷新动作条映射方可恢复。
6. 在 HUD 手动点击或直接鼠标点击原生动作条后，确认短暂 `manual_hold` 优先于任何同帧后续派发：动作码与 BindingToken 均为 `0`。主键启停、爆发顺序、脱战 Burst 硬门控、CD 数字、DurationObject 转盘、标签和排序不得变化。
