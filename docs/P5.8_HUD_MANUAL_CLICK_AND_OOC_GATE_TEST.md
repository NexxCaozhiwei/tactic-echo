# P5.8 HUD 点击与脱战爆发硬门控实机验证

1. 脱战、HUD 显示官方爆发窗口时，观察 `/temapping` / SavedVariables：不得出现 `pre_window_capture_started`、`plan_created`、`step_dispatched`、`precombat_burst_bridge_*` 或 Burst TEAP/TEK 派发。
2. 脱战时依次点击 HUD 主键启停：只改变启动/暂停显示；不得使爆发序列派发。
3. 将客户端“按键按下即施放（ActionButtonUseKeyDown）”分别设为开启、关闭，各测试爆发、打断、控制、防御/生存 HUD 卡。每次都应与点击其原始动作条按钮效果一致，且记录 `manual_click_priority:hud`。
4. 在相同时间点触发一个普通/爆发推荐后，先点击 HUD 卡或原生动作条按钮：该帧/短暂让权窗口必须为 `manual_hold`，动作码与 BindingToken 为 0，后续才允许普通派发。
5. 把 13/14 槽物品分别放在不同默认动作条按钮或已有 `/use 13`、`/use 14` 宏上：HUD 饰品卡必须执行对应 slot 的原有按钮；无可靠来源时只显示不可点击原因。
6. 战斗中切换动作条页、隐藏按钮或改变宏来源：HUD 卡应保持显示但禁止执行并提示“需要脱战后重建 HUD 点击映射”；脱战后 `/tecache refresh` 或动作条更新后恢复。
