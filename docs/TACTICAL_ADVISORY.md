# 战术建议层与队列 HUD

## 防御

防御建议由兼容低血布尔值，或“近期受伤 + 新鲜高压环境信号”触发。按紧急/大减伤/小减伤/自疗排列；未在现实动作条找到绑定时可以显示“无绑定”原因，但永不进入派发。

## 爆发窗口

Burst Window Helper 使用 `IDLE / ARMED / ACTIVE / COOLDOWN / SUPPRESSED / UNKNOWN` 状态，仅显示窗口与候选。当前首批 Profile 为惩戒骑、冰法、猫德。它不向官方推荐队列注入技能，也不会自动使用饰品 13/14、药水、种族技能或团队增益。

## 打断、控制、位移

- 打断基于事件派生的目标读条状态；已发生打断后短暂抑制重复提示。
- 明确不可打断的读条才展示控制候选。
- 位移由低血或新鲜环境布尔信号触发；环境未知时显示未知，不伪装为识别了地板机制。

## 队列 HUD

- 官方主推荐：唯一可能进入既有 TEAP/TEK 派发链路的图标。
- 候选、打断、防御、爆发、控制、位移：仅预览/提示。
- `可派发 / 已阻止 / 状态未知 / 无绑定 / 暂停` 都是显示语义，不改变安全判定。
- 默认横向布局可切换为纵向或环绕；防御队列可独立拖动、缩放、锁定和保存位置。

HUD 模型固定槽位、内容防抖、单图标 fail-soft。它绝不读官方 API、写 Token、编码 TEAP 或请求 TEK 输入。
