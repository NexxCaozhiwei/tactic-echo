# P5.1 宏动作身份实机验收

## 目标

验证反制射击宏改名为中文“打断”、英文“DDD”或其他名称时，均能被识别；同名但不同技能宏不得干扰。

## 准备

将以下宏放在当前可见 Blizzard 默认动作条，并绑定支持的实体键位：

```lua
#showtooltip
/stopcasting
/cast [@focus,nodead] 反制射击
/cleartarget
/targetenemy
/cast 反制射击
/targetlasttarget
```

测试宏名依次改为：

```text
打断
DDD
反射
```

可额外创建同名宏，例如另一枚“打断”宏内容为：

```lua
/cast 奥术射击
```

它用于验证同名但不同技能不会被选中。

## 验收步骤

1. 覆盖 AddOn 后 `/reload`。
2. 执行：

```text
/tecache refresh
/temapping p51_macro_action_identity
```

3. 打开自动打断映射。反制射击应显示为该宏的焦点/当前目标路由，三种宏名结果一致。
4. 保存退出后检查 `TacticEchoDB.diagnostics.mappingExport` 对应宏：
   - 若 `actionInfoId=147362` 一类代表技能值：
     - `actionInfoLooksLikeMacroIndex=false`
     - `representedSpellID=147362`
     - `macroIdentitySource=action_text_unique_spell_semantic`
     - `semanticCandidateCount=1`
   - 若客户端返回真实宏 index：
     - `actionInfoLooksLikeMacroIndex=true`
     - `macroIdentitySource=action_info_macro_index`
5. 创建两枚同名、且都引用反制射击的宏时，必须不识别自动路由，并记录：

```text
macro_semantic_identity_ambiguous
```

这是预期安全行为。

## 不在本验收范围

- 不改变钢条/可打断资格；
- 不修改焦点、当前目标或鼠标指向宏分支；
- 不改变 TEK，不需要重建 `TEK.exe`。
