# P5.10 当前动作栏宏身份与冰冻陷阱 HUD 实机验证

## 准备

使用猎人，在当前可见 Blizzard 默认动作条中放置一个既有宏，例如：

```text
#showtooltip
/cast [@cursor] 冰冻陷阱
```

把该宏绑定在 `CTRL+1`。可在 `CTRL+2` 放置冰冻陷阱直放或另一枚冰冻陷阱宏，用来验证不会错误回退。不要让 Tactic Echo 新建、复制或修改任何宏。

## 验收步骤

1. `/reload` 后确认控制 HUD 能识别冰冻陷阱。来源应指向当前 `CTRL+1` 的按钮与动作栏槽位，而不是其它同技能位置。
2. 点击控制 HUD。实际结果应完全等同于鼠标点击 `CTRL+1` 对应的原生动作条宏；`[@cursor]` 的落点行为必须由玩家原宏保持。不得自动切换目标、不得生成新按键，也不得出现 TEAP/TEK 控制派发。
3. 保留 `CTRL+2` 的同技能按钮，将 `CTRL+1` 替换为冰冻陷阱直放、其他宏或移出当前可见栏位。在战斗中 HUD 必须保持显示但阻断，提示“原动作条来源已变化”或“身份无法唯一确认”；不得改为点击 `CTRL+2`。
4. 脱战后刷新动作条映射，再将原 `[@cursor]` 宏放回 `CTRL+1`。HUD 应重新恢复为仅复用该原按钮。
5. 创建两枚不同宏名、但正文都引用冰冻陷阱的候选，使动作条显示名与 action-info handle 名分别能匹配不同候选。HUD 必须不执行，诊断应为 `macro_semantic_identity_ambiguous`；不得任选其一。
6. 验证正文失配：让唯一同名宏正文改为 `/cast 威慑`，即使 `GetMacroSpell` 或 action-info 仍报告冰冻陷阱，控制卡也不能获得可靠手动来源。
7. 回归：主键启停、HUD CD/转盘/标签/排序正常；HUD 与原生默认动作条的真实左键仍产生 `manual_hold`，动作码与 BindingToken 为 0；自动打断仍返回 `auto_interrupt_suspended`；任何脱战帧都不产生 Burst plan/capture/TEAP/TEK。
