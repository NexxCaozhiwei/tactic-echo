# Tactic Echo 1.0.37 基线：不透明自身 CD 的原生倒计时受控回退

## 本版范围

1.0.37 以 1.0.36 的稳定 HUD 实时 `DurationObject` 路径为基础，只修复一个显示末级回退：客户端已公开“技能自身、非共享 GCD 冷却进行中”，却暂时没有可传递给 HUD 的安全数字时，自定义 HUD 模式此前同时隐藏原生数字与自定义数字，形成转盘/数字空白或错误诊断。

本版不改 AutoBurst、有序步骤计划、简易/集中预检、GCDGate、四帧 handoff、起手 bridge、切图围栏、TEAP 或 TEK。

## 冷却展示合同

```text
安全数字可读
→ 单一最终 DurationObject 画转盘
→ HUD 自定义数字按模块样式显示

安全数字暂不可读，但已确认非 GCD 自身 CD
→ 单一最终技能专用 / 已认证直接动作条 DurationObject
→ 临时显示该对象的原生倒计时
→ 数字恢复后自动回到 HUD 自定义样式

纯共享 GCD / 61304 别名 / 物品泛 GCD
→ 不显示为技能自身 CD
```

- 通用动作条 `DurationObject` 必须先有同一可信直接按钮的安全可读非 GCD 自身 CD 数值证据；无此证据时不得用于普通技能、窗口、注入、打断或防御卡。
- 覆盖/变体技能的 `matchedSpellID` 与动作条自身 CD 证据必须参与 HUD 冷却渲染签名，防止 reload、动作条刷新或覆盖变化时沿用旧对象。
- Tooltip 文案必须区分“临时原生倒计时”和“仅转盘、等待安全数字”；不得再笼统声称“原生转盘和倒计时渲染”。

## 验收重点

1. 对窗口、注入、打断/控制、防御分别制造“CD active、数值可读”和“CD active、数值暂不可读”的短暂阶段；前者显示 HUD 自定义数字，后者只临时显示一套同对象原生数字，不能空白或双层。
2. 主键/窗口纯 GCD、饰品泛 GCD 均不显示 CD 数字或错误自身转盘。
3. `/reload` 后等待动作条/技能 API 重建期间，CD 来源从不透明状态恢复到 `actionbar_numeric_observed` 或可信动作条数值时，显示应平滑切回 HUD 样式。
4. 已验证的 AutoBurst 简易/集中序列行为必须与 1.0.36 一致。
