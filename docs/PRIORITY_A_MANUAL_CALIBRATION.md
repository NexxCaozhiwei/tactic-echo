# 实机校准与验收清单

自动测试已覆盖协议、缓存、派发安全、宏诊断、战术边界和队列 HUD；以下项目只能在 Windows/WoW 实机确认。

## Windows 与输入

- [ ] 单屏 100%、125%、150% DPI。
- [ ] 双显示器、负坐标副屏和主副屏切换。
- [ ] WoW 窗口化、无边框、全屏。
- [ ] Notepad 单键、组合键、滚轮验收。
- [ ] WoW 前台可输入、非前台绝不输入。
- [ ] 文本框、聊天框、宏编辑、按键设置打开时暂停。
- [ ] 玩家真实键盘/鼠标输入期间 TEK 让权；恢复后仅接受新帧。

## 动作条与宏

- [ ] 默认动作条主栏、多栏、分页、Bonus Bar、姿态/潜行/德鲁伊形态。
- [ ] 额外动作按钮存在时主推荐仍可解析。
- [ ] 载具、控制单位、覆盖动作条、宠物战斗硬阻断。
- [ ] 单行无条件 `/cast` 宏映射。
- [ ] 条件、目标修饰、多行、`/use`、`/castsequence` 宏只核对诊断；除非 Blizzard 当前宏 API 明确报告对应法术，否则不应由正文推断映射。

## 图标、爆发与队列 HUD

按 `惩戒骑 → 冰法 → 猫德` 校验：冷却、充能、资源、距离、触发、施法/引导、目标状态和未知字段 Tooltip。

- [ ] 主推荐、候选、打断、防御、爆发、控制、位移的边框、透明度、状态遮罩符合设置。
- [ ] `/tehud horizontal`、`vertical`、`surround` 三种布局的图标锚点不重叠；候选内容切换不明显跳动。
- [ ] 拖动官方主推荐图标可移动主 HUD；拖动独立抓手可移动拆分防御队列；锁定后均不可移动。
- [ ] 独立防御队列的缩放、透明度和位置在 `/reload` 后保持。
- [ ] 冷却扇形、充能边缘环、快捷键、层数和当前施法标记正常显示。
- [ ] 出现 secret-number 时只显示 `?` 或“未知”，没有 Lua 错误，也不会隐藏整个 HUD。
- [ ] 防御兼容低血与高压信号只产生建议，不改变主推荐。
- [ ] 打断、不可打断控制、位移环境提示只在对应事件/新鲜信号下出现。
- [ ] Burst Window Helper 只显示状态/候选，不生成额外 TEAP Token 或 TEK 输入。
- [ ] 全职业候选表未校准的专精明确显示为职业级候选，不作为实战准确性承诺。
