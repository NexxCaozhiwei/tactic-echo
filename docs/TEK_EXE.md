# 0.9.51 P5.6 客户区自动定位

TEK 自动模式不要求在 WoW 全屏时操作状态窗口。它从当前 WoW HWND 获取客户区坐标，只在左上 `360×96 → 640×160 → 最大 960×240 px` 区域寻找 v3 的 `T / E / 3` 头部。候选布局需要两帧完整 20 格 CRC/Commit 校验，并要求 freshness 前进；失败时保持等待，绝不改用桌面 `(12,12)` 坐标。

AddOn 把相同 v3 信号放在 UIParent 左上、8px 安全边距；格子按有效 UI 缩放确保至少约 8 个物理像素。运行期只截取已验证条带并使用 3×3/5×5 中位数采样。独占全屏若截图后端拿到黑帧或旧桌面帧，属于捕获兼容性问题，不应被当作“校准失败”。

# TEK.exe 运行说明

## 0.9.11 色块采样可靠性

Pillow 是运行期默认采样后端；它一次读取同一桌面图像，再由 TEAP 的 RGB 冗余、Commit 与 CRC16 校验有效帧。GDI 仅为“连接”页可选实验模式；连续解码失败时会自动回退到 Pillow，并保持到本次 TEK 退出。自动定位失败采用退避，退避期间改用已配置的固定坐标，避免反复全屏扫描。状态窗口的“采样后端”和“色块采样诊断”会显示当前后端、回退状态和最近解码错误。

## 0.9.10 遥测稳定化

`tek-status.json` 现由后台发布器维护：状态变化异步排队，稳定状态最多每 250 ms 写入。状态文件锁定或权限错误仅触发后台重试和限频日志，不会终止 TEK worker。普通 trace 已精简；“高级诊断”开关开启时才额外采集完整前台、色块、限流和物理输入详情。

## 0.9.40 本地白名单与输入门禁

TEK 的输入链路必须同时满足：TEAP CRC/协议/目录校验、已武装状态、帧新鲜度、WoW 前台一致性、非文本输入、非 `channeling` / `empowering` / `manual_hold` 状态、键盘 Hook 健康和全局限频。

真实键盘的人工接管只依赖 TEK 的“连发介入”本地白名单：

- 默认白名单：`W`、`A`、`S`、`D`、`SPACE`。白名单键按住或松开均不进入人工接管恢复流程。
- 白名单外的真实键盘输入按住时持续阻断 SendInput；全部释放后才开始恢复。
- `CTRL`、`ALT`、`SHIFT`、`WIN` 不能作为独立白名单项保存；组合键是否豁免由主键决定。
- 鼠标左/右/侧键、中键、滚轮、拖拽和鼠标移动全部忽略，不影响 TEK 连发。
- balanced 恢复策略：`30 ms + 2 个新的 TEAP freshness 帧 + 150 ms 同技能/同动作 replay guard`。

键盘 Hook 停止时，TEK 显示“等待 Hook 停止”并拒绝新工作会话；只有确认旧 Hook 线程退出后才能再次启动。真实 `windows_sendinput` 仅要求键盘低级 Hook 健康；鼠标不属于该门禁。Windows Hook、WoW 前台、实际 SendInput 与多屏/DPI 仍需人工实机验收；离线测试不能替代该验收。

## 诊断

TEK 可生成脱敏诊断包，包含设置、状态、近期 trace、运行日志、显示/DPI 信息、映射快照和 `SUMMARY.md`。它不复制完整 SavedVariables，也不导出宏正文。
