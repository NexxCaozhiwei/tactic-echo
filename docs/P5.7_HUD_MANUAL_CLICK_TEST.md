# P5.7 实机验收：HUD 手动点击与自动打断暂停

1. `/reload` 后打开“打断”页：**自动打断（当前不可用）**应可见、未勾选且不可点击。旧 SavedVariables 即使曾开启，也应显示 `auto_interrupt_suspended`，不得出现 reaction candidate。
2. 主 HUD 图标左键：应在“启动 / 暂停”之间切换，行为与原控制面板按钮一致；拖动 HUD 时不应切换。
3. 在脱战状态，将一个可用爆发、打断、控制或防御技能放在暴雪默认动作条（或已被 Resolver 识别的既有宏）上：对应 HUD 图标悬停应显示“HUD 手动点击：可用”；左键应触发与原动作条完全相同的技能/宏。
4. 移除来源、隐藏动作条、进入特殊动作条，或在战斗中改变动作条页/宏来源：HUD 卡片仍显示，但左键必须不执行，并显示原因。脱战后刷新 HUD 应恢复可用映射。
5. 在存在官方/Burst 推荐时左键 HUD 技能，或用鼠标左键点击原生动作条技能：随后短暂 TEAP 状态应为 `manual_hold`，动作码和 BindingToken 均为 0；不得追加 TEK 派发。
6. 核对 HUD CD 数字、转盘、状态标签、来源标签、爆发顺序和技能排序，与 P5.6 基线一致。
## 自动打断历史集成测试

P5.7 将自动打断硬暂停，因此原先以 `AutoReaction:Evaluate()` 生成 `candidate` 为成功条件的历史集成测试会标记为跳过。它们保留在仓库中作为路由/宏识别行为的历史契约，且不会为测试建立任何隐藏的生产环境重新启用开关。当前生效的运行时测试应验证：即使旧 SavedVariables、正向事件、可用绑定和可打断读条同时存在，结果仍只能是 `auto_interrupt_suspended`。
