# P4.6 反制射击多行宏识别实机验收

## 宏

将以下宏放在**当前可见的 Blizzard 默认动作条**，并分配受支持的真实键位：

```lua
#showtooltip
/stopcasting
/cast [@focus,nodead] 反制射击
/cleartarget
/targetenemy
/cast 反制射击
/targetlasttarget
```

这是一枚玩家原有的焦点优先、选敌回退、恢复原目标宏。TE 只会在通过既有 P4 资格后按一次该宏的已绑定键位；宏内 `/cleartarget`、`/targetenemy`、`/targetlasttarget` 始终由 WoW 宏引擎执行。

## 预期识别

1. 覆盖 AddOn 并 `/reload`。
2. 打开自动打断设置中的 P2 映射区域。
3. `反制射击` 应显示为已识别宏，且 `focus` 与 `target` 路由共享同一个真实键位；说明文字包含“焦点优先、选敌回退并恢复原目标的宏”。
4. 该结果不要求当前正在战斗、也不要求存在读条。

## 未识别时的采集

1. 依次执行：

```text
/tecache refresh
/temapping p46_countershot
```

2. 正常退出游戏，使 SavedVariables 落盘。
3. 从 `TacticEcho.lua` 的 `diagnostics.mappingExport.entries` 中查找对应宏按钮。P4.6 应提供以下无正文诊断字段：

```text
macroLookupSource
macroLookupAttemptCount
macroLookupByActionText
macroLookupByActionInfoName
macroActionInfoName
macroResolvedSpellTokenCount
macroFailureReason
```

4. 判读：

| 现象 | 含义 |
|---|---|
| `macroLookupByActionInfoName=true` 且 `macroResolvedSpellTokenCount=2` | 已通过动作条宏名回收正文，并将两条 `/cast 反制射击` 解析为同一 SpellID；应进入 P2 宏路由。 |
| `macroFailureReason=macro_action_text_and_name_missing` | 客户端动作条既未给宏文本也未给宏索引名，无法安全关联。 |
| `macroFailureReason=macro_lookup_failed_after_action_info_name` | 索引名称已取得，但未能在既有宏列表找到正文；检查是否在宏列表中存在同名/异常宏。 |
| `macroResolvedSpellTokenCount=0` 且正文长度大于 0 | 本地化名称/SpellID API 当时仍无法解析；请保留导出供下一轮诊断。 |

## 动作验证

识别成功后，仍按既有 P4.4 规则验证可打断读条与真实键位；P4.6 只修复宏映射发现，不放宽钢条/可打断证据，也不修改 TEK。
