# Tactic Echo 1.0.26 基线：可排序多步骤爆发序列

## 范围

- 每专精独立保存窗口、最多 3 个注入技能、饰品 13/14 的启用状态与顺序。
- 顺序以稳定身份 `window`、`injection:<SpellID>`、`trinket:13|14` 保存，不保存动作条键位、瞬时 ItemID 或注入槽位编号。
- 窗口固定启用；注入与饰品可上移、下移、启用、停用。

## 计划组装合同

1. 官方推荐精确命中当前专精窗口时，先读取所有已启用可选步骤的真实绑定和实时 CD。
2. `simple`：排除明确自身 CD、UNKNOWN、无绑定、未装备或装备身份不匹配步骤；其余步骤严格保持用户顺序。
3. `focused`：任一已启用可选步骤不可用即不创建计划，官方窗口保持普通路径。
4. 若所有可选步骤被排除，不创建计划；窗口不被 Burst 夺取。
5. 共享 GCD / QUEUE_WINDOW / GCD_LOCKED 不等于自身 CD，仍可作为可用步骤入队。
6. 每个计划步骤派发前再次读取实时状态。简易模式跳过运行期失效的当前可选步骤；集中模式在窗口尚未派发时释放官方窗口，窗口已派发后才持有离开锁。

## Handoff 与起手

- 仅当实际筛选后的第一步位于窗口之前时，才进行四帧 observation-only handoff。
- 已 arm 的前置序列可使用既有受控起手 bridge 跨越脱战→进战；普通脱战官方推荐仍不可派发。
- 窗口第一的序列不创建脱战 bridge，也不会扩大普通脱战派发权限。
