# Tactic Echo 1.0.23 基线：配置表默认窗口 / 注入映射

## 目标

将 `BurstProfiles.lua` 中以下专精的 Profile fallback 默认规则与用户提供的《部分窗口注入技能配置表》逐项对齐。配置表中的“窗口技能”仅作为官方推荐的精确窗口锚点；“注入技能”仅作为该窗口前/后可由现有 AutoBurst 规则插入的受控步骤。

## 范围与不变项

- 仅更新 `BurstProfiles` 的默认窗口 / 注入 seed；不重置用户已有的当前专精启用、禁用、排序或自定义技能设置。
- 手动 `autoBurstWindowSpellID` / `autoBurstInjectionSpellID` 仍高于 Profile fallback；用户显式写入的手动规则不会被本次覆盖。
- 不修改 TEAP v3、TEK 安全门禁、1.0.20 四帧 handoff、1.0.21 起手 Burst bridge、1.0.22 动作条 CD 跳过逻辑。
- `REFERENCE_DURATION_SEEDS` 未被配置表授权变更，保留现有值；它只服务于独立 HUD 爆发状态机的 fallback，不参与 AutoBurst 步骤派发顺序。

## 回归要求

1. 上述 7 个 Profile fallback 分别解析为精确的单一窗口 SpellID 与单一注入 SpellID；
2. 旧值不得继续作为该 7 个专精的默认 seed；
3. 未列入配置表的专精默认配置保持不变；
4. 手动规则优先级与已有用户专精列表不退化。
