# Tactic Echo 1.0.06 基线

## 修复目标

本版本修复自动爆发第一阶段前置注入的严格顺序语义：

```text
窗口技能成为官方推荐
→ 注入技能已确认自身可用
→ 仅允许 Burst 候选派发注入技能
→ 注入技能确认成功
→ 才允许 Burst 候选派发窗口技能
```

## 核心修复

- AutoBurst 的逐步技能冷却采样改为实时公共 API 读取，不再复用 HUD 的事件缓存。
- GCD 快照始终实时读取；事件缓存不再被当作 GCD 调度时钟。
- 实时 API 已标记为共享 GCD 时，CooldownTracker 的旧缓存不得覆盖这一事实。
- 前置计划在注入技能已于窗口边沿确认可用后，获得窗口所有权；后续瞬态状态、确认超时或计划中止不得把窗口技能交还普通官方派发链。
- 前置注入确认失败时终止并保持窗口离开锁；不会出现“4 没有成功，1 却由官方链派发”的行为。
- 后置简易仍允许窗口成功后跳过失败的可选注入；该例外不适用于前置序列。

## 实机重点

使用处决宣判（`343527`，键位 `1`）作为窗口、复仇之怒（`31884`，键位 `4`）作为注入，前置简易模式下：

1. 在窗口推荐边沿且复仇之怒自身可用时，诊断必须先出现 `dispatch_origin=burst`、`binding=4`。
2. 在复仇之怒确认前，不得出现官方来源的 `binding=1`。
3. 复仇之怒确认后，才允许 Burst 来源派发 `binding=1`。
4. 若 4 的候选或确认失败，1 应被观察锁抑制直到官方推荐离开该窗口，不得绕过序列。
